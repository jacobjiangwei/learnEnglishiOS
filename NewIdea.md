【2026-02-08 更新】我对功能和 UX 的新理解

原则:
- 之前写的模块代码都保留, 不删除
- 当前仅本地查词功能保留, 但降级处理(作为基础能力而非核心入口)
- UX 优先于功能堆叠, 先解决新手上手和等级匹配的问题

一、核心 UX 调整

1) 新人 Onboarding: 先定级, 再学习
- 新人进入先选择目标等级: KET / PET / 初一 / 初二 / 初三 / 高中 / 大学 / 研究生 / 日常
- 用户自选等级只是初始意向, 不是最终定级

2) 等级能力测评
- 进入等级测试(10-20 题)
- 通过: 进入该等级
- 未通过: 提示是否降级(更容易通过的等级), 引导继续测试
- 测评结果决定主学习路径和题目难度

3) 使用一段时间后触发限制, 引导登录与付费
- 前期允许游客模式体验
- 达到阈值后限制(例如每日题量/复习次数/功能入口)
- 弹出提示: 登录账号并订阅, 解锁无限服务

4) 本地查词功能保留但降级
- 查词作为辅助入口, 不再作为核心学习起点
- 重点入口改为: 等级学习 / 练习 / 复习 / 测评

二、需要加入的能力清单

- Onboarding 流程(选择等级 + 测评)
- 等级测评题库(基础题库 + AI 动态生成)
- 降级提示逻辑(未通过测评时)
- 游客模式 + 付费解锁
- 登录系统(先海外: Apple/Google, 后国内: 微信/手机号)

三、现有代码处理策略

- 查词、生词本、复习功能保留
- 调整入口优先级和导航结构, 让 Onboarding 和测评成为主流程
- 后续再逐步恢复情景对话、写作等高阶模块
